**Please add support for the `CUBE` clause as described in the SQL:1999/2003 standard.**

`CUBE` is a syntactic shortcut defined in the SQL standard for generating **all possible combinations of subtotals and a grand total** based on the grouping columns specified. It is logically equivalent to a complete set of `GROUPING SETS`. Its primary purpose is to simplify complex cross-tabulation and summary analysis.

-----

### Key Characteristics According to the SQL Standard

The `CUBE` operation generates every possible subtotal for the grouping columns. For N columns, `CUBE` generates 2^N aggregation levels (e.g., `CUBE(A, B)` generates 2^2=4 groupings).

  * **Complete Cross-Tabulation:** `CUBE(A, B)` generates the following groupings:
    (A, B), (A), (B), ()
  * **Syntax:**
    ```sql
    SELECT 
        column_A, 
        column_B, 
        Aggregate_Function(column_C)
    FROM 
        TableName
    GROUP BY
        CUBE (column_A, column_B); -- Equivalent to GROUPING SETS ((A, B), (A), (B), ())
    ```

### The `GROUPING` Function

The standard **`GROUPING(column)`** function must be supported to distinguish summary rows generated by `CUBE` from true detail rows.

  * **Output:** `GROUPING(column)` returns **1** if the column is currently being aggregated out (holding a summary `NULL` value) and **0** if the column is part of the current grouping set.

-----

## Example using the `mtcars` Dataset

We will use `CUBE` to calculate the **Total Horsepower (`hp`)** based on all four possible combinations of `cyl` and `gear`.

**Goal:** Calculate HP totals for every combination: (`cyl`, `gear`), (`cyl`), (`gear`), and Grand Total.

```sql
SELECT
    cyl,
    gear,
    SUM(hp) AS Total_HP,
    -- Use GROUPING to label the subtotals
    CASE
        WHEN GROUPING(cyl) = 0 AND GROUPING(gear) = 0 THEN 'Detail Subtotal'
        WHEN GROUPING(cyl) = 0 THEN 'CYLINDER TOTAL'
        WHEN GROUPING(gear) = 0 THEN 'GEAR TOTAL'
        ELSE 'GRAND TOTAL'
    END AS Group_Level
FROM
    mtcars
GROUP BY
    CUBE (cyl, gear) -- Generates: (cyl, gear), (cyl), (gear), ()
ORDER BY
    cyl NULLS LAST, gear NULLS LAST;
```

## Implementation Requirements

### 1\. Core Feature Mapping

  * The implementation must recognize **`CUBE (col1, col2, ...)`** syntax within the `GROUP BY` clause.
  * The `CUBE` operation **must be internally translated** into the equivalent, standard set of `GROUPING SETS` before execution.
      * Example: `CUBE(A, B)` -> `GROUPING SETS((A, B), (A), (B), ())`

### 2\. `GROUPING` Function Support

  * The output must correctly populate the summary rows with `NULL` for the cubed columns.
  * The `GROUPING(column)` function must correctly return **1** for summary `NULL` values generated by the `CUBE` process.

### 3\. Testing Requirements

  * Create tests to verify the functionality in a test class called **`jparq.group.CubeTest`**.
  * Tests must cover:
      * Simple `CUBE(A, B)` and verification of the 2^N result levels.
      * Correct calculation of totals/subtotals.
      * Correct output of the `GROUPING(column)` function across all rows.
  * Adhere to all coding standards (Javadoc, Checkstyle, PMD, Spotless).
  * All existing tests must pass after implementation using `mvn -Dspotless.check.skip=true verify`.
